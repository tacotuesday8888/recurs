# Base Engine Comparison

This review compares Recurs with the current open-source engines behind Kilo Code, OpenCode, and Codex. The goal is not feature parity. It is to identify the smallest set of engine guarantees Recurs needs before the sub-agent architecture is built on top.

Reviewed on 2026-07-10 through the GitHub connector:

- [Kilo Code](https://github.com/Kilo-Org/kilocode/tree/8324cf7ddc6539993f7d1743175716ae2705d195)
- [OpenCode](https://github.com/anomalyco/opencode/tree/8a03fc265b6d73c2e15881fcc702c9cb3027dd0e)
- [Codex](https://github.com/openai/codex/tree/dc5ae378967cff0de2cfb30b98c52047ab978e3d)

Parallel scheduling was rechecked on 2026-07-19 against current [Codex](https://github.com/openai/codex/blob/0fb559f0f6e231a88ac02ea002d3ecd248e2b515/codex-rs/core/src/tools/parallel.rs), which uses an explicit per-tool capability behind a shared/exclusive execution gate, and [Gemini CLI](https://github.com/google-gemini/gemini-cli/blob/acae7124bdd849e554eaa5e090199a0cf08cd782/packages/core/src/scheduler/scheduler.ts), which batches contiguous calls with an explicit ordering barrier.

Stream liveness was rechecked on 2026-07-20 against current [Codex provider configuration](https://github.com/openai/codex/blob/main/codex-rs/core/config.schema.json), which exposes a streaming-response idle timeout and bounded stream retries. Recurs applies one smaller provider-neutral deadline to normalized direct-provider events so the same safety boundary covers every implemented adapter and compaction path.

Tool-argument enforcement was rechecked on 2026-07-21 against [Codex's closed Serde tool structs](https://github.com/openai/codex/blob/d5998e74522245ce6e7b746dab6c5f0ed428a8d1/codex-rs/protocol/src/plan_tool.rs), [OpenCode's central runtime decoder](https://github.com/anomalyco/opencode/blob/849c2598abc7d2b40261e74b5826bc74ffc78308/packages/opencode/src/tool/tool.ts), and [Gemini CLI's validate-before-invocation boundary](https://github.com/google-gemini/gemini-cli/blob/acae7124bdd849e554eaa5e090199a0cf08cd782/packages/core/src/tools/tools.ts). Recurs follows that established baseline: the schema shown to a model is also enforced locally before permission evaluation or effects, while tool-specific semantic checks remain explicit in each concise parser.

OpenAI Responses transport behavior was rechecked on 2026-07-20 against current [Codex session transport state](https://github.com/openai/codex/blob/main/codex-rs/core/src/client.rs) and the official [Responses WebSocket guide](https://developers.openai.com/api/docs/guides/websocket-mode). Both transports use the same response-event ordering. Recurs reuses one authenticated socket across a run's sequential tool rounds, but switches to HTTPS/SSE only when the WebSocket handshake fails before `response.create` is sent; an interruption after delivery fails without replay.

Gemini transport behavior was rechecked on 2026-07-21 against Google's official [GenerateContent API](https://ai.google.dev/api/generate-content), [Models API](https://ai.google.dev/api/models), and [thought-signature guidance](https://ai.google.dev/gemini-api/docs/generate-content/thought-signatures). Recurs implements the smaller stateless Developer API path: authenticated model discovery, bounded SSE, native text/image/function content, normalized usage, and exact process-scoped signature replay across tool rounds. It deliberately does not claim Vertex AI, the newer Interactions API, or cross-process recovery of an unfinished signed tool turn.

MCP lifecycle was rechecked on 2026-07-19 against current [Codex](https://github.com/openai/codex/blob/0fb559f0f6e231a88ac02ea002d3ecd248e2b515/codex-rs/core/src/session/mcp.rs), which retains a session-owned manager when the effective server projection is unchanged, current [Gemini CLI](https://github.com/google-gemini/gemini-cli/blob/acae7124bdd849e554eaa5e090199a0cf08cd782/packages/core/src/tools/mcp-client.ts), which keeps successful server connections and exposes their state, and the [MCP lifecycle](https://modelcontextprotocol.io/specification/2025-11-25/basic/lifecycle), which defines initialization, per-request timeout/cancellation, ping, and transport shutdown responsibilities.

Project instructions were rechecked on 2026-07-19 against current [Codex](https://github.com/openai/codex/blob/0fb559f0f6e231a88ac02ea002d3ecd248e2b515/codex-rs/core/src/agents_md.rs), which discovers `AGENTS.md` from the nearest configured project root through the cwd, prefers an override filename, and applies one aggregate byte budget. Recurs adopts the hierarchy and bounded turn context but rejects symlinks, invalid UTF-8, mid-read changes, and over-limit aggregates instead of following or truncating them.

Long-running commands were rechecked on 2026-07-20 against current [Codex unified exec](https://github.com/openai/codex/blob/678157acaa819d5510adfe359abb5d0392cfe461/codex-rs/core/src/tools/handlers/shell_spec.rs), [Gemini CLI shell sessions](https://github.com/google-gemini/gemini-cli/blob/acae7124bdd849e554eaa5e090199a0cf08cd782/docs/tools/shell.md), and [Kilo's interactive-terminal boundary](https://github.com/Kilo-Org/kilocode/blob/938919ab72e3977d1512e0363417270e3337c7b1/packages/opencode/src/kilocode/tool/interactive-terminal.txt). These projects distinguish ordinary commands, background or resumable processes, and user-driven PTY sessions rather than treating every shell call as one blocking operation.

User clarification was rechecked on 2026-07-20 against current [Codex request-user-input protocol](https://github.com/openai/codex/blob/main/codex-rs/app-server/README.md#request_user_input) and [Gemini CLI's ask-user tool](https://github.com/google-gemini/gemini-cli/blob/main/docs/reference/tools.md#interaction). Both model an in-turn question as a host-mediated interaction rather than an ordinary assistant message or a workspace permission.

Repository intelligence was rechecked on 2026-07-21 against [Aider's repository map](https://github.com/Aider-AI/aider/blob/5dc949bdc38f2a812086227dffc18680d9596734/aider/repomap.py), [OpenCode's LSP manager](https://github.com/anomalyco/opencode/blob/45cd8d76920839e4a7b6b931c4e26b52e1495636/packages/opencode/src/lsp/lsp.ts), and [Gemini CLI's codebase investigator](https://github.com/google-gemini/gemini-cli/blob/acae7124bdd849e554eaa5e090199a0cf08cd782/packages/core/src/agents/codebase-investigator.ts). Aider builds parser-backed ranked maps, OpenCode manages optional language-server processes, and Gemini composes bounded read/search primitives through a read-only agent. Recurs uses the official TypeScript compiler syntax tree for TypeScript/JavaScript declarations and retains bounded lexical extraction for its other supported languages. Its fixed `typescript_diagnostics` tool now gives the root agent real no-emit project diagnostics in Plan or Act mode; cross-file reference ranking remains explicitly lexical, existing child profiles remain version-stable, and Recurs does not claim a persistent language server or general semantic navigation.

File discovery was rechecked against current [OpenCode glob](https://github.com/anomalyco/opencode/blob/45cd8d76920839e4a7b6b931c4e26b52e1495636/packages/opencode/src/tool/glob.ts) and [Gemini CLI glob](https://github.com/google-gemini/gemini-cli/blob/acae7124bdd849e554eaa5e090199a0cf08cd782/packages/core/src/tools/glob.ts). Recurs keeps its smaller `list_files` surface and optional bounded glob, preserving every existing child profile while retaining literal argv execution, repository ignore behavior, and deny-last credential exclusions. It closes an additional local boundary by collecting NUL-delimited path bytes, omitting non-UTF-8 and credential-classified records, sorting deterministically by UTF-8 bytes, and rendering JSON records so control characters in legal filenames cannot alter result framing.

Repository search was rechecked on 2026-07-21 against current [OpenCode grep](https://github.com/anomalyco/opencode/blob/dev/packages/opencode/src/tool/grep.ts) and [Gemini CLI filesystem tools](https://github.com/google-gemini/gemini-cli/blob/main/docs/tools/file-system.md). Both expose regular-expression search with a path/include boundary and bounded results. Recurs keeps fixed text as the safe compatibility default, adds regex only through an explicit mode, and returns normalized JSON match records so paths and control characters cannot be confused with result framing. Query, glob, per-file collection, raw process output, rendered output, and returned-result count are independently bounded; credential exclusions remain deny-last.

Git workspace evidence was rechecked on 2026-07-21 against the official [`git status` porcelain format](https://git-scm.com/docs/git-status). Git documents porcelain v2 as a stable machine format and recommends `-z` when machine parsing is required. Recurs consumes that byte protocol directly, validates its branch and change records, and renders bounded JSON lines; it does not parse the human-oriented short-status display. Raw review patches remain a separate `git_diff` surface with literal path selection.

Public web evidence was rechecked on 2026-07-20 against [OpenCode's `webfetch`](https://github.com/anomalyco/opencode/blob/45cd8d76920839e4a7b6b931c4e26b52e1495636/packages/opencode/src/tool/webfetch.ts), [Gemini CLI's `web_fetch`](https://github.com/google-gemini/gemini-cli/blob/acae7124bdd849e554eaa5e090199a0cf08cd782/packages/core/src/tools/web-fetch.ts) and [network helpers](https://github.com/google-gemini/gemini-cli/blob/acae7124bdd849e554eaa5e090199a0cf08cd782/packages/core/src/utils/fetch.ts), and [Codex's hosted web-search policy](https://github.com/openai/codex/blob/0fb559f0f6e231a88ac02ea002d3ecd248e2b515/codex-rs/core/src/web_search.rs). OpenCode and Gemini establish permissioned, bounded textual fetches; Codex keeps hosted search policy distinct from local tools. Recurs adds a smaller known-URL tool and closes its local transport boundary by verifying and socket-pinning every DNS hop, rejecting mixed answers and cross-origin redirects, and labeling all returned text as untrusted.

Conversation branching was rechecked on 2026-07-20 against [Codex `thread/fork`](https://github.com/openai/codex/blob/0fb559f0f6e231a88ac02ea002d3ecd248e2b515/codex-rs/app-server/README.md#threads) and [OpenCode's session fork endpoint](https://github.com/anomalyco/opencode/blob/45cd8d76920839e4a7b6b931c4e26b52e1495636/packages/opencode/src/server/routes/instance/httpapi/groups/session.ts). Recurs implements the smaller safe CLI primitive: fork one idle direct-provider conversation at its exact durable sequence. It carries visible context and immutable policy, but deliberately resets branch-local accounting and authority and rejects opaque runtime continuations.

Interactive model selection was rechecked on 2026-07-20 against [Gemini CLI's `/model` command](https://github.com/google-gemini/gemini-cli/blob/main/docs/cli/model.md) and [OpenCode's explicit provider/model CLI options](https://github.com/anomalyco/opencode/blob/dev/packages/web/src/content/docs/cli.mdx). Recurs keeps selection tied to durable saved connections: `/model` shows exact provider, model, billing, primary, and execution metadata, and a confirmed selection creates a new immutable session rather than redirecting existing history or silently changing the primary.

Reasoning selection was rechecked on 2026-07-20 against [Codex app-server model metadata](https://github.com/openai/codex/blob/main/codex-rs/app-server/README.md) and [OpenAI's current model guidance](https://developers.openai.com/api/docs/guides/latest-model). Codex can present Ultra as a product effort tier, while the reviewed GPT-5.6 Responses API uses `max` as its highest wire value. Recurs preserves that distinction instead of translating an unsupported label into a request.

Headless prompt input was rechecked on 2026-07-21 against current [Codex exec](https://github.com/openai/codex/blob/main/codex-rs/exec/src/lib.rs), which supports forced or explicitly appended stdin, and the [Gemini CLI reference](https://github.com/google-gemini/gemini-cli/blob/main/docs/cli/cli-reference.md), which documents piped prompt input. Recurs keeps the explicit forms and adds one fixed byte and UTF-8 boundary before a session is created.

Image prompt input was rechecked on 2026-07-21 against the [Codex CLI image workflow](https://learn.chatgpt.com/docs/prompting#cli-workflow-image--prompt), [Gemini CLI multimodal file injection](https://github.com/google-gemini/gemini-cli/blob/main/docs/cli/custom-commands.md), and [OpenCode's documented image attachment](https://opencode.ai/en/docs). Recurs uses an explicit repeatable headless flag and the same bounded contract through ACP, then validates the file before durable storage or provider dispatch instead of relying on terminal drag-and-drop semantics.

Working-root selection was rechecked on 2026-07-21 against current [Codex CLI options](https://github.com/openai/codex), which expose `-C`/`--cd` for one working root, and [Gemini CLI's workspace model](https://github.com/google-gemini/gemini-cli/blob/main/docs/tools/file-system.md), which roots filesystem tools in the workspace while allowing separately configured additional directories. Recurs adopts the single explicit root and deliberately omits extra writable roots so session identity, project policy, MCP trust, and sandbox containment share one canonical boundary.

CLI discoverability was rechecked on 2026-07-21 against current [OpenCode CLI flags](https://opencode.ai/docs/cli/) and [Gemini CLI options](https://github.com/google-gemini/gemini-cli/blob/main/docs/cli/cli-reference.md), which both expose ordinary help and version contracts. Recurs now prints the exact build version without runtime initialization, provides scoped help for each public command group, and reuses that one build identity in ACP and MCP protocol metadata.

Kilo's CLI is a fork of OpenCode, as Kilo states in its [README](https://github.com/Kilo-Org/kilocode/blob/8324cf7ddc6539993f7d1743175716ae2705d195/README.md). They are therefore one engine lineage for this comparison; Kilo-specific guards are considered separately.

## Recurs base assessment

The current code is a real harness, not a placeholder. Beyond the original provider-neutral loop, tools, permissions, Plan mode, goals, sessions, compaction, checkpoints, structured output, cancellation, and loop guards, it now has reviewed provider onboarding, immutable backend pins, bounded root-to-cwd project instructions, direct/delegated coordination, ACP/MCP/skills interoperability, owned long-running commands, owned child and durable team execution, and fail-closed macOS/Linux tool containment.

Its useful difference is size. Recurs can keep the single-agent engine understandable while placing future complexity in provider adapters and the sub-agent orchestration layer.

The original four hardening requirements are now implemented:

1. One strict provider-event reducer shared by tests and the live loop.
2. Durable cleanup and restart reconciliation for every tool call.
3. A same-session run guard and an honest platform-specific permission/containment boundary.
4. A single local/CI verification command plus concise current documentation.

## Explicitly deferred

The remaining material gaps are narrower and explicit:

- Windows OS-level filesystem/network sandboxing and a Linux seccomp layer (macOS uses fail-closed Seatbelt; Linux uses fail-closed system Bubblewrap namespaces and mount policy);
- richer full-screen terminal/scrollback management, arbitrary signal controls, and safe hidden input for authentication/passphrase prompts; the local user-present CLI now has bounded raw PTY attach/detach, but deliberately does not claim those additional UI and secret-capture capabilities;
- parser-backed declarations beyond TypeScript/JavaScript, semantic reference resolution, persistent language-server features, and non-TypeScript project diagnostics (the bounded mixed-engine `code_outline` map and fixed no-emit TypeScript project checking are implemented);
- a reviewed general plugin lifecycle, remote MCP/OAuth, and desktop UI;
- additional live provider-specific transports beyond the current local, fixed-origin Chat, public OpenAI Responses/Anthropic Messages/Gemini GenerateContent, Codex ACP, and private native OpenAI/Anthropic/Kimi paths;
- additional authoritative limit discovery for local, native, delegated-runtime, and ephemeral routes, plus child pre-turn compaction; the current automatic path is deliberately limited to parent sessions created from saved BYOK connections with authenticated setup-time limits and real provider usage;
- transport fallback for providers other than the public OpenAI Responses path, plus Codex's hooks, connectors, and mature multi-agent protocol.

Windows containment and a reviewed Linux syscall policy remain important follow-ups before Recurs allows unattended arbitrary-command workers. The standalone macOS and Linux defaults now add fail-closed OS boundaries while retaining permission prompts as a separate decision layer.
