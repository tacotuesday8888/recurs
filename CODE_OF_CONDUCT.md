# Code of Conduct

Recurs should be a place where people can do careful technical work without
being demeaned, harassed, or pressured to disclose private information.

## Expected behavior

- Discuss ideas and evidence, not a person's worth or motives.
- Give specific, actionable feedback and accept correction in good faith.
- Respect different experience levels, identities, backgrounds, and
  communication styles.
- Protect credentials, private code, security reports, and personal data.
- Stop an interaction when someone sets a reasonable boundary.

Harassment, threats, discriminatory language, sexualized attention, deliberate
intimidation, doxxing, impersonation, and sustained disruption are not
acceptable in project spaces.

## Scope and enforcement

This standard applies in repository issues, pull requests, reviews, and other
spaces where someone is representing the project. Maintainers may edit or
remove content, close or lock threads, reject contributions, or temporarily or
permanently restrict participation when needed to protect the community.

Report a conduct concern privately to the repository maintainer through a
channel you already share. Include only the context needed to evaluate it. If
no safe private channel exists, use GitHub's own block and abuse-reporting
controls; do not publish sensitive details to create a report.

Maintainers should minimize disclosure, avoid conflicts of interest, and
explain enforcement decisions to the affected person when doing so is safe.
